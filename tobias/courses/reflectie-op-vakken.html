Smart Connection


Toepassing tijdens het project
Bij Smart Connection hebben we leren programmeren. Als ICT'er heb ik bij dit vak helaas niets nieuws geleerd, maar ik heb mijn kennis kunnen toepassen bij het project door een business case tool te programmeren.

Tops
  • Variatie in opdrachten: ik vond het goed dat naast normale html en css ook met JavaScript data uit een API geladen moest worden.
  • Tempo: het tempo waarmee we door de stof gaan is denk ik perfect voor beginners.

Tips
  • Volgorde van lessen veranderen: in de eerste lessen worden de talen c++, javascript en html behandeld. Dit is voor beginners erg ingewikkeld. Ik zou c++ bij het vak Smart Technology behandelen en alleen HTML bij Smart Connection in de eeste week. Net voor de API opdrachten zou ik pas JavaScript behandelen.
  • Voeg meer uitdaging toe voor ICT'ers


Smart Technology


Toepassing tijdens het project
Bij Smart Technology hebben leren 3D modelleren, dit hebben we kunnen toepassen bij het project. We hebben een bestaande mal voor gaasmatten kunnen aanpassen in SolidWorks om hem geschikt te maken voor Wopereis.
Ook hebben we bij dit vak geleerd hoe we een IoT schakeling kunnen maken, dit hadden we ook graag willen toepassen tijdens het project maar hier zijn we helaas niet aan toegekomen. We hadden een monitoringskastje willen bouwen voor de robot om te kijken hoe lang per dag de robot gemiddeld aan staat. Tijdens dit vak hebben we geleerd hoe we dit zouden kunnen doen.

Tops
  • 3D-printen: persoonlijk vond ik dit het leukste onderdeel van de minor.
  • Compleet vak: ik vind het goed dat we veel verschillende dingen hebben behandeld namelijk programmeren, PCB ontwerp, solderen en 3d-modelleren. Dit vak op zichzelf biedt ons als studenten voldoende kennis om zelfstandig hele toffe dingen te maken.


Tips
  • Code standaard verbeteren: als ICT'er is er niets zo vervelend als chaotische code bekijken. Toch gebeurde het vaak wanneer ik klasgenoten wilde helpen dat ze al hun c++ code voor de IoT schakeling in een bestand hadden staan. Het opsplitsen van code zou daarom behandeld moeten worden, niet alleen omdat ik vind dat chaotische code vervelend is om te lezen, maar ook omdat het dan veel overzichterlijker wordt voor beginners. Ik denk ook echt dat een uitleg van 5 minuten al genoeg is om het opsplitsen van code duidelijk te maken. De setup() en loop() worden veel beter leesbaar als er enkel een aantal functieaanroepen in staan.
